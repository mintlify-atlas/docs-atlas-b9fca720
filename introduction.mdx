## What is Zeal?

Zeal is an offline documentation browser for software developers. It provides instant, offline access to documentation for 200+ programming languages, frameworks, and tools. Inspired by [Dash](https://kapeli.com/dash) for macOS, Zeal brings powerful documentation browsing to Windows, Linux, and macOS.

## How It Works

Zeal uses **docsets** — compressed documentation archives that contain HTML documentation, search indexes, and metadata. These docsets are compatible with the Dash format, giving you access to a vast ecosystem of pre-built documentation sets.

<Steps>
  <Step title="Download Zeal">
    Install Zeal on your operating system from the official downloads page or package manager.
  </Step>
  <Step title="Download Docsets">
    Open Zeal and navigate to **Tools → Docsets**. Select the documentation sets you want (Python, JavaScript, C++, etc.) and download them.
  </Step>
  <Step title="Search and Browse">
    Use the search bar to find documentation instantly. Press the global shortcut to access Zeal from anywhere.
  </Step>
</Steps>

## Quick Example

Search for documentation across multiple docsets:

```bash
# General search
python:pprint

# Search specific docsets
java:BaseDAO

# Search multiple docsets
python,django:string
```

You can also launch Zeal with a query from the command line:

```bash
zeal python:pprint
```

## Platform Support

Zeal is available on all major desktop platforms:

- **Windows** — Installer and portable versions available
- **Linux** — Available via package managers (APT, Flatpak, AppImage)
- **macOS** — Homebrew and direct downloads

<Note>
  Zeal requires Qt 5.15.2 or higher and uses Qt WebEngine for rendering documentation.
</Note>

## Architecture

Zeal is built with modern C++ and Qt, organized into modular components:

- **Core** — Application lifecycle, settings, and HTTP server
- **Registry** — Docset management and search indexing
- **Browser** — WebEngine-based documentation viewer
- **UI** — Main window, dialogs, and sidebar components

Learn more in the [Architecture Overview](/development/architecture/overview).
