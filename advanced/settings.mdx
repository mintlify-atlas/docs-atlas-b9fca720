---
title: Settings
description: Configure Zeal's behavior, appearance, and network settings
---

Zeal stores its settings in a configuration file that can be edited manually or through the application's settings dialog.

## Configuration File Location

<Tabs>
  <Tab title="Standard Build">
    Settings are stored in the platform-specific location:
    
    - **Linux**: `~/.config/Zeal/Zeal.conf`
    - **Windows**: `%APPDATA%\Zeal\Zeal.ini`
    - **macOS**: `~/Library/Preferences/org.zealdocs.Zeal.plist`
  </Tab>
  <Tab title="Portable Build">
    Settings are stored in `zeal.ini` in the application directory.
  </Tab>
</Tabs>

## Settings Categories

### Startup

Control how Zeal behaves when launched.

<ParamField path="startMinimized" type="boolean" default="false">
  Start Zeal minimized to the system tray
</ParamField>

<ParamField path="checkForUpdate" type="boolean" default="true">
  Automatically check for application updates
</ParamField>

<ParamField path="ui.hide_menu_bar" type="boolean" default="false">
  Hide the menu bar (can be temporarily shown with Alt key)
</ParamField>

### System Tray

Configure system tray icon and window behavior.

<ParamField path="show_systray_icon" type="boolean" default="true">
  Display Zeal icon in the system tray
</ParamField>

<ParamField path="minimize_to_systray" type="boolean" default="false">
  Minimize to system tray instead of taskbar
</ParamField>

<ParamField path="hide_on_close" type="boolean" default="false">
  Hide window instead of closing when clicking the close button
</ParamField>

### Global Shortcuts

Set keyboard shortcuts that work system-wide.

<ParamField path="global_shortcuts.show" type="string">
  Global keyboard shortcut to show/hide Zeal window
  
  Example: `Alt+Space`, `Meta+Z`
</ParamField>

### Tabs

Control tab behavior.

<ParamField path="tabs.open_new_tab_after_active" type="boolean" default="false">
  Open new tabs after the currently active tab instead of at the end
</ParamField>

### Search

Configure search behavior.

<ParamField path="search.fuzzy_search_enabled" type="boolean" default="false">
  Enable fuzzy search matching
  
  When enabled, search results don't require exact character matches in sequence.
</ParamField>

### Content

Customize documentation appearance and fonts.

#### Appearance

<ParamField path="content.appearance" type="enum" default="Automatic">
  Control the appearance theme
  
  - `Automatic` (0) - Follow system theme
  - `Light` (1) - Always use light theme
  - `Dark` (2) - Always use dark theme
</ParamField>

#### Fonts

<ParamField path="content.default_font_family" type="string" default="serif">
  Default font family for documentation
  
  Options: `serif`, `sans-serif`, `monospace`
</ParamField>

<ParamField path="content.serif_font_family" type="string">
  Serif font family name (e.g., "Times New Roman")
</ParamField>

<ParamField path="content.sans_serif_font_family" type="string">
  Sans-serif font family name (e.g., "Arial")
</ParamField>

<ParamField path="content.fixed_font_family" type="string">
  Fixed-width font family name for code (e.g., "Courier New")
</ParamField>

<ParamField path="content.default_font_size" type="integer" default="16">
  Default font size in pixels
</ParamField>

<ParamField path="content.default_fixed_font_size" type="integer" default="13">
  Default fixed-width font size in pixels
</ParamField>

<ParamField path="content.minimum_font_size" type="integer" default="0">
  Minimum font size in pixels
</ParamField>

#### Behavior

<ParamField path="content.highlight_on_navigate" type="boolean" default="true">
  Highlight search term when navigating to a result
</ParamField>

<ParamField path="content.smooth_scrolling" type="boolean" default="true">
  Enable smooth scrolling animation
</ParamField>

<ParamField path="content.custom_css_file" type="string">
  Path to custom CSS file for styling documentation
</ParamField>

<ParamField path="content.external_link_policy" type="enum" default="Ask">
  How to handle external links
  
  - `Ask` (0) - Prompt user before opening
  - `Open` (1) - Open in embedded browser
  - `OpenInSystemBrowser` (2) - Open in system browser
</ParamField>

### Network

Configure network and proxy settings.

#### Proxy

<ParamField path="proxy.type" type="enum" default="System">
  Proxy configuration type
  
  - `None` (0) - No proxy
  - `System` (1) - Use system proxy settings
  - `Http` (3) - HTTP proxy
  - `Socks5` (4) - SOCKS5 proxy
</ParamField>

<ParamField path="proxy.host" type="string">
  Proxy server hostname or IP address
</ParamField>

<ParamField path="proxy.port" type="integer">
  Proxy server port number
</ParamField>

<ParamField path="proxy.authenticate" type="boolean" default="false">
  Enable proxy authentication
</ParamField>

<ParamField path="proxy.username" type="string">
  Proxy authentication username
</ParamField>

<ParamField path="proxy.password" type="string">
  Proxy authentication password
</ParamField>

<ParamField path="proxy.ignore_ssl_errors" type="boolean" default="false">
  Ignore SSL certificate errors
  
  <Warning>
    Only enable this if you understand the security implications.
  </Warning>
</ParamField>

### Docsets

<ParamField path="docsets.path" type="string">
  Path to docset storage directory
  
  **Default locations:**
  - Linux: `~/.local/share/Zeal/docsets`
  - Windows: `%LOCALAPPDATA%\Zeal\docsets`
  - macOS: `~/Library/Application Support/Zeal/docsets`
  
  For portable builds, defaults to `docsets` in the application directory.
</ParamField>

## Example Configuration

Here's an example configuration file snippet:

```ini
[General]
start_minimized=false
check_for_update=true
show_systray_icon=true
minimize_to_systray=false
hide_on_close=false

[ui]
hide_menu_bar=false

[global_shortcuts]
show=Alt+Space

[search]
fuzzy_search_enabled=true

[content]
appearance=0
default_font_family=serif
default_font_size=16
default_fixed_font_size=13
minimum_font_size=0
external_link_policy=0
highlight_on_navigate=true
smooth_scrolling=true

[proxy]
type=1
ignore_ssl_errors=false

[docsets]
path=/home/user/.local/share/Zeal/docsets
```

## Internal Settings

Zeal also maintains some internal settings that should not be manually edited:

- `internal.install_id` - Unique UUID identifying the installation
- `internal.version` - Configuration format version
- `state.window_geometry` - Window size and position
- `state.splitter_geometry` - Splitter positions
- `state.toc_splitter_state` - Table of contents splitter state

<Note>
  Settings are automatically saved when you close Zeal or change preferences in the settings dialog.
</Note>
